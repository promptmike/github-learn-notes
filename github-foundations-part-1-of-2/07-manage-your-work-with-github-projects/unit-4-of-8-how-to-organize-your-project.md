## Create a field to track and group by priority

![New Field](https://learn.microsoft.com/en-us/training/github/manage-work-github-projects/media/4-project-new-field.png)
![Drop-down](https://learn.microsoft.com/en-us/training/github/manage-work-github-projects/media/4-new-field-drop-down.png)

1. Rightmost field header > Select `+`
2. Select *New field*
3. Type a field name
4. Select *Field Type* dropdown
5. Select a classification, or *Single select* to create your own
6. Add priority levels to the *Options* box
7. Select *Save*

> Projects offers field creation with custom classications and priorities for grouping and tracking by priority

Use the field row *Priority*, select an issue or pull request, then use the dropdown to select a priority level for the item
> Projects offers field creation, tracking and grouping, multiple views and prioritisation

**Group items by priority**
1. Open view > Select the dropdown arrow
2. Select *Group by*
3. Select *Priority*

> Projects allows you to create custom fields, track items, group by priority and use multiple views

When items are grouped by priority, you can drag and drop them into new priority fields with the UI
> Projects offers intuitive GUI to create fields, track items, and group by priority

![Priority Classification](https://learn.microsoft.com/en-us/training/github/manage-work-github-projects/media/4-priority-classification-example.png)

Click save changes in the top-right to save a list view
> Projects offers intuitive GUI to create fields, track and group items, and reset priorities with simple drag-and-drop

## Add an iteration field

Iteration fields allow you to set phases for tasks
> With projects, you can create fields, track and group items, set priorities, edit views with drag-and-drop and use iterations to plan work

![Iteration Field](https://learn.microsoft.com/en-us/training/github/manage-work-github-projects/media/4-iteration-field.png)

1. Go to project table view
2. Rightmost header > Select `+`
3. Select *New field*
4. Add a field name
5. Field type > Select *Iteration*
6. Choose start date
7. Set durations by typing numbers and selecting either *days* or *weeks* from the drop-down
8. Select *Save and create*

> Projects offers custom fields, track and group, custom priorities, drag-and-drop views, and configurable iterations with project phases

You can assign your issues and pull requests to iteration phases
> Projects offers custom fields and views, track and group, drag-and-drop interface for views, and iterations to group items by project phase

## Create a board view

1. Open view and select the down arrow
2. Go to *Layout* > Select *Board*
3. Top-right > Select *Save*
4. Double click the view tab to rename
5. Alternatively, Select *New View* (right of views)

> Projects offers custom fields, track and group, drag-and-drop interfaces, iteration for project phases, and multiple views to work with your project

You can drag and drop items into columns from the board view
> Projects offers custom fields, tracking, grouping by priority, iteration for phases, and drag-and-drop interfaces for views

![Project Boards](https://learn.microsoft.com/en-us/training/github/manage-work-github-projects/media/4-project-boards.png)

## Project workflows

GitHub Projects includes built-in workflows for common actions
> Projects offers custom fields, tracking, grouping by custom priorities, iteration of project phases, drag-and-drop views, and built-in workflows for common actions

**Example: Set initial status of new issues to "Todo"**
1. Open a project
2. Select `...` > *Workflows*
3. Select trigger (in this case, *Item added to project*)
4. Configure values (in this case, *Status: Todo"*)
5. *Save and turn on workflow*

> GitHub Projects offers custom fields, tracking, grouping by custom priorities, iteration of project phases, drag-and-drop views, and workflows for automation, including built-ins
